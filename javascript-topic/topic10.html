JS Events

JavaScript events are actions or occurrences that happen in the browser, typically as a result of user interactions or system triggers. These events enable developers to create interactive and dynamic web applications. Here's a discussion along with some examples of JavaScript events:

  1. Event Handling
    • Event handling involves defining code (event handlers) that should execute in response to specific events.
    • In JavaScript, event handlers are functions that are invoked when an event occurs.

    2. Types of Events

  2. Mouse Events:
    • These events are triggered by mouse interactions such as clicks, hover, mouse movement, etc.

    3. Example Code:

    document.getElementById("myButton").addEventListener("click", function() {
    console.log("Button clicked!");
});

    Output

  3. Keyboard Events
    • These events are triggered by keyboard interactions such as key presses and releases.

    4. Example Code:

    document.addEventListener("keydown", function(event) {
    console.log("Key pressed: " + event.key);
});

    Output:

    Press any key: 

    (you will see the result by inspect element and go to console)
  4. Form Events
    • These events are triggered by interactions with HTML form elements such as submitting a form, changing input values, etc.

    5. Example Code:

    document.getElementById("myForm").addEventListener("submit", function(event) {
    event.preventDefault(); // Prevent form submission
    console.log("Form submitted!");
});

    Output:

    (you will see the result by inspect element and go to console)
  5. Document and Window Events
    • These events are triggered by changes in the document or window state, such as loading, resizing, scrolling, etc.

    6. Example Code:

    window.addEventListener("load", function() {
    console.log("Page loaded!");
});

    Output:

    Scroll down:

    (you will see the result by inspect element and go to console)
    Event Registration
    • Event handlers can be registered to HTML elements using event attributes (onclick, onmouseover, etc.) or by using the addEventListener() method.
    • Using addEventListener() allows for attaching multiple event handlers to the same event on an element and provides more flexibility.
    Event Object
    • When an event occurs, the browser creates an event object that contains information about the event (e.g., the type of event, target element, mouse coordinates, etc.).
    • Event objects are passed as arguments to event handler functions, allowing developers to access event-related data.

    Example Code:

    document.getElementById("myElement").addEventListener("click", function(event) {
    console.log("Clicked on element: " + event.target.id);
});

    Output:

    Click Me 

    (you will see the result by inspect element and go to console)
    Event Propagation
    • JavaScript events follow a bubbling or capturing propagation model, where an event triggered on a specific element can also affect its parent and ancestor elements (bubbling) or its child and descendant elements (capturing).
    • Developers can control event propagation using methods like stopPropagation() and preventDefault().

    Example Code:

    document.getElementById("myButton").addEventListener("click", function(event) {
    event.stopPropagation(); // Stop event from bubbling up
    console.log("Button clicked!");
});

    (you will see the result by inspect element and go to console)

JavaScript events is crucial for creating interactive and responsive web applications. By leveraging events, developers can build engaging user experiences and enhance the interactivity of their web applications.
